# 점검 이슈 번호 체계 정리 작업

## 배경

현재 roadmap.md "🔥 점검 발견 이슈" 섹션의 번호 체계가 두 개 섞여 있어 혼란 발생:
- **체계 A**: 마스터 P0 번호 (#15, #16, #17, #18, #19, #21, #22, #23)
- **체계 B**: 점검 종합의 자체 번호 (#1~#18)

특히 #15, #16, #17, #18이 카테고리 간 중복.

## 새 번호 체계

카테고리별 prefix 도입:
- Critical → `C-1, C-2, ...`
- Important → `I-1, I-2, ...`
- Minor → `M-1, M-2, ...`

## 작업
