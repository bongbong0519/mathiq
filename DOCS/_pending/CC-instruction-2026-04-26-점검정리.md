### 작업 #16: 운영자 승인 대기 탭 활성화

**증상**: 회원 현황 → 승인 대기 탭에 "선생님 인증 시스템은 준비 중입니다" 표시. 신규 가입자가 안 보임.

**현재 상태**: 선생님 탭에서는 대기중 상태로 보이고 승인 가능. 즉, 데이터는 있는데 승인 대기 탭이 비활성.

**작업**:

3. 승인 대기 탭에서도 [승인][거절] 버튼 작동
4. 원장님/학생/학부모 탭의 동일 패턴도 점검

**수락 기준**: 신규 가입자가 승인 대기 탭에 자동 표시. 거기서 직접 승인 가능.

### 작업 #17: 외래키 ON DELETE CASCADE 적용 + 사용자 삭제 함수

**증상**: profiles/auth.users 참조 외래키 37개에 CASCADE 없음. 사용자 삭제 시 DB error.

**작업 — 두 트랙 병행**:

**트랙 A: ON DELETE CASCADE 적용 (단순)**
- 37개 외래키에 ALTER TABLE로 CASCADE 추가
- 단, 일부 테이블(예: account_deletions, recovery_requests)은 soft delete 의도라 CASCADE 부적합 → 별도 처리 필요

**트랙 B: 영구삭제 RPC 함수 (안전)**
- `delete_user_permanently(user_id)` 함수 작성
- 37개 테이블 순차 정리 → 마지막에 auth.users 삭제
- 30일 cron 트리거에서 호출 가능하도록

**작업 순서**:
1. 외래키 37개 분류 (CASCADE OK / 함수로 처리할 것)
2. CASCADE 일괄 적용 (마이그레이션 파일)
3. delete_user_permanently RPC 작성
4. 테스트 데이터로 검증

**수락 기준**: 운영자가 사용자 강제 삭제 가능. 30일 자동 영구삭제 트리거가 안전 작동.
