## P0 처리

### 자물쇠 모달 닫기 ✅

**문제**: X/닫기/업그레이드/ESC 모두 작동 안 함

**원인**: 
- `showTierLockModal()`: `modal.style.display = 'flex'` 사용
- `closeModal()`: `classList.remove('open')` 사용
- 불일치!

**수정**:
```javascript
function closeModal(id) {
  const el = document.getElementById(id);
  if (el) {
    el.classList.remove('open');
    el.style.display = 'none';  // 추가
  }
}
```

**추가**:
- 배경 클릭: `onclick="closeModal('tierLockModal')"`
- ESC 키: `document.addEventListener('keydown', ...)`

**검증**: X/닫기/ESC/배경클릭 모두 ✅

## 외래키 정리

### 진단 결과

profiles/auth.users 참조 외래키 약 37개 (CASCADE 미적용)

### 막던 진짜 원인 (실제 검증)

1. **recovery_requests** (2행) - user_id 참조
2. **account_deletions** - user_id 참조
3. **posts** - author_id 참조
4. **auth.identities** - Supabase 내장 테이블

### 정리 과정

```
profiles 18명 삭제 시도
→ recovery_requests 막음 → 2행 삭제
→ account_deletions 막음 → 정리
→ profiles 삭제 성공 (1명 남음)

auth.users 8명 삭제 시도
→ posts 막음 → author_id != admin 삭제
→ auth.identities 막음 → user_id != admin 삭제
→ auth.users 삭제 성공 (1명 남음)
```

### 작성된 SQL 파일
- `_pending/cleanup-test-users-v2-2026-04-26.sql`
- `_pending/cleanup-auth-users-orphans.sql`
- `_pending/migration-017-cascade-delete.sql`

### C-1 본 작업 (베타 후)

**방향 1**: ON DELETE CASCADE 일괄 적용
- 대부분 테이블에 적용
- account_deletions, recovery_requests → SET NULL

**방향 2**: RPC 함수
- `delete_user_permanently(user_id)`
- 37개 테이블 순차 정리 → auth.users 삭제
- pg_cron으로 30일 자동 실행

### DB 사용자 참조 컬럼 지도 (2026-04-27 새벽 진단)

총 37개 (뷰 5개 제외 = 실제 32개):

| 컬럼 | 개수 | 테이블 |
|-----|:---:|-------|
| teacher_id | 12 | accounting_expense, accounting_income, accounting_settings, billing_invoices, calendar_events, class_materials, classes, exams, expenses, payment_reminders, students, teacher_comments, uploaded_files |
| user_id | 10 | account_deletions, ai_usage, feedback_votes, notice_reads, privacy_consents, recovery_requests, students, teacher_payment_settings, tutee_profiles, tutor_profiles |
| author_id | 6 | feedback_comments, feedback_posts, inquiries, notices, post_comments, posts |
| owner_id | 4 | academy_staff, accounting_staff_salary, organizations, v_monthly_salary |

**특이점:**
- expenses + accounting_expense 동시 존재 (구/신 구조?)
- students에 teacher_id + user_id 둘 다 (선생님 + 학생 본인 계정)
