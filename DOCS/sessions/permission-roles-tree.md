# 직원 형태 분류 트리

> 작성: 2026-04-27  
> 출처: 권한 매트릭스 (19개 직원 형태 × 12개 기능)  
> 용도: Canvas 마인드맵 + 권한 프리셋 시스템 설계 기준

## 🌳 분류 트리

```
직원 형태 (19)
│
├─ A. 강사 (8)
│  ├─ A-1. 핵심강사 [패턴 TEACHER_A]
│  │  ├─ 정규직 강사
│  │  ├─ 프리랜서 강사
│  │  └─ 인적용역 강사
│  │
│  ├─ A-2. 일반강사 [패턴 TEACHER_B]
│  │  ├─ 시간제 강사
│  │  └─ 외국인 강사
│  │
│  ├─ A-3. 제한강사
│  │  ├─ 초단시간 강사 [패턴 TEACHER_C]
│  │  └─ 보조 강사     [패턴 TEACHER_D]
│  │
│  └─ A-4. 최소강사 [패턴 TEACHER_E]
│     └─ 일용 강사
│
├─ B. 운영/행정 (3)
│  ├─ B-1. 관리자급 [패턴 FULL]
│  │  └─ 부원장
│  │
│  └─ B-2. 행정직
│     ├─ 실장 (데스크) [패턴 STAFF_FRONT]
│     └─ 조교 (행정 보조) [패턴 STAFF_ASSIST]
│
└─ C. 비권한자 / 미정 (8)
   ├─ C-1. 시스템 내 제한 접근
   │  ├─ 인턴·실습생 [패턴 STAFF_ASSIST — 조교와 동일]
   │  └─ 차량 [패턴 EXTERNAL — 캘린더만]
   │
   ├─ C-2. 권한 없음 [패턴 NONE]
   │  ├─ 청소·시설
   │  └─ 외주 전문가
   │
   └─ C-3. 별도 정의 필요 [패턴 TBD]
      ├─ 배우자/자녀/친족
      ├─ 위탁운영 인력
      └─ 기타 (직접 입력)
```

## 📌 핵심 발견

### 권한 패턴 중복

19개 직원 형태 중 **고유 권한 패턴은 9개 + TBD 3개**.

| 패턴 ID | 해당 직원 형태 | 비고 |
|---------|--------------|------|
| `FULL` | 부원장 | 12개 기능 전권 |
| `TEACHER_A` | 정규직, 프리랜서, 인적용역 | 학부모문자 ✅ |
| `TEACHER_B` | 시간제, 외국인 | 학부모문자 ❌ |
| `TEACHER_C` | 초단시간 | AI유사문제 ❌, 문제등록 ❌ |
| `TEACHER_D` | 보조 | 시험관리 ❌, AI유사문제 ❌, 문제등록 ❌ |
| `TEACHER_E` | 일용 | 캘린더만 + 대시보드/학생관리 |
| `STAFF_FRONT` | 실장 | 회계 ✅, 학부모문자 ✅ |
| `STAFF_ASSIST` | 조교, 인턴·실습생 | 대시보드/학생관리/수업자료/캘린더만 |
| `EXTERNAL` | 차량 | 캘린더만 |
| `NONE` | 청소·시설, 외주 전문가 | 시스템 접근 없음 |
| `TBD` | 배우자/자녀/친족, 위탁운영, 기타 | 케이스별 정의 필요 |

### 시사점

- **228칸 (19×12)을 일일이 관리할 필요 없음** → 9개 프리셋 + 직원에 프리셋 매핑
- 세무/근로계약상으로는 다른 카테고리(예: 정규직 vs 프리랜서)도 **시스템 권한상으로는 동일**할 수 있음 → DB 분리 안 해도 됨
- **C-3 (배우자/자녀/친족, 위탁운영, 기타)**는 베타 후 케이스 정리 필요
