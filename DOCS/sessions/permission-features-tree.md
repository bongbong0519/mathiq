## 🌳 분류 트리

```
기능 (12)
│
├─ 1. 학생/학부모 영역 (3)
│  ├─ 학생관리      [공통접근, A-4·C 제외 거의 ✅]
│  ├─ 성적리포트    [A-3 보조 이상]
│  └─ 학부모문자    [⚠️ 분기점 — A-1·B-1·실장만]
│
├─ 2. 학습/평가 영역 (5)
│  ├─ 시험관리      [A-3 보조부터 ❌, A-2 이상만 ✅]
│  ├─ AI유사문제    [⚠️ 분기점 — 보조·초단시간 ❌]
│  ├─ 문제은행      [A-3 보조 이상]
│  ├─ 문제등록      [⚠️ 분기점 — 보조·초단시간 ❌]
│  └─ 수업자료      [A-4 일용 제외 거의 모두]
│
├─ 3. 공통 영역 (2)
│  ├─ 대시보드      [거의 모두 ✅, C-2/C-3만 ❌]
│  └─ 캘린더        [차량까지도 ✅, C-2/C-3만 ❌]
│
└─ 4. 관리자 영역 (2)
   ├─ 회계         [⚠️ 분기점 — 부원장·실장만]
   └─ 강사관리     [⚠️ 분기점 — 부원장만 (전권 표식)]
```

---

## 📌 권한 분기점 (Critical Branches)

직원 형태를 가르는 결정적 기능들:

| 기능 | 분기 의미 |
|------|---------|
| **학부모문자** | A-1 핵심강사 vs A-2 일반강사 |
| **AI유사문제** | A-2 이상 vs A-3 제한강사 |
| **문제등록** | A-2 이상 vs A-3 제한강사 |
| **시험관리** | A-3 초단시간 vs A-3 보조 |
| **회계** | 실장 이상 (행정 권한 표식) |
| **강사관리** | 부원장 전용 (최고권한 표식) |

→ 이 6개 기능만 보면 직원 권한 레벨이 결정됨

---

## 🔓 공통 기능 (거의 전원 ✅)

`대시보드`, `캘린더`, `학생관리`, `수업자료` — 강사 + 행정직은 모두 접근.

→ **권한 시스템 설계 시 "기본 접근권"으로 묶을 수 있음**.

---

## 🔒 최고권한 기능 (부원장만 ✅)

`강사관리` — 매트릭스에서 부원장만 ✅인 유일한 기능.

→ **"관리자 모드" 토글의 기준 기능**으로 사용 가능.

## 💡 시스템 설계 힌트

### 권한 그룹화

12개 기능을 시스템에서 다루기 좋은 단위로 묶으면:

```typescript
// 예시 구조
type PermissionGroup =
  | "STUDENT_BASIC"      // 학생관리 + 성적리포트
  | "STUDENT_CONTACT"    // 학부모문자
  | "EXAM_BASIC"         // 시험관리 + 문제은행 + 수업자료
  | "EXAM_ADVANCED"      // AI유사문제 + 문제등록
  | "COMMON"             // 대시보드 + 캘린더
  | "ADMIN_FINANCE"      // 회계
  | "ADMIN_STAFF"        // 강사관리
```

### 프리셋 매핑 예시

```typescript
const TEACHER_A = ["STUDENT_BASIC", "STUDENT_CONTACT", "EXAM_BASIC", "EXAM_ADVANCED", "COMMON"]
const TEACHER_B = ["STUDENT_BASIC",                     "EXAM_BASIC", "EXAM_ADVANCED", "COMMON"]
const TEACHER_C = ["STUDENT_BASIC",                     "EXAM_BASIC",                  "COMMON"]
// ... 등
const FULL      = ["STUDENT_BASIC", "STUDENT_CONTACT", "EXAM_BASIC", "EXAM_ADVANCED", "COMMON", "ADMIN_FINANCE", "ADMIN_STAFF"]
```

→ **228칸 매트릭스 → 7개 그룹 × 9개 프리셋**으로 단순화 가능
