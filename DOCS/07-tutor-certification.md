# 선생님 인증 시스템 설계

> **문서 상태**: 설계 완료, 구현 예정  
> **작성일**: 2026-04-21  
> **구현 목표**: Phase 1 (3~6개월 후)

---

## 1. 목적 및 배경

### 왜 선생님 인증이 필요한가?

MathIQ는 "투명한 사교육"을 비전으로 합니다. 현재 과외 시장의 문제점:

| 문제 | 현황 | MathIQ 해결책 |
|------|------|---------------|
| 정보 비대칭 | 학부모는 선생님 실력 검증 어려움 | 객관적 인증 등급 제공 |
| 무자격 강사 | 신고 없이 불법 과외 성행 | 자격증 검증 필수화 |
| 우수 선생님 불이익 | 자격 있어도 차별화 어려움 | 등급별 인센티브 |
| 사회적 비용 | 탈세, 불법 과외 문제 | 합법 선생님만 노출 |

### 기대 효과

- 학부모: "MathIQ 인증 = 합법 + 검증"이라는 신뢰
- 선생님: 자격 있는 분이 정당한 평가 받음
- 사회: 사교육 시장 투명화 기여

---

## 2. 핵심 원칙

1. **선생님 본인이 직접 등록**
   - 학부모가 대신 등록 불가
   - 본인 인증 필수 (휴대폰/이메일)

2. **자격증 검증 필수**
   - 개인과외교습자 신고필증 또는 학원강사 등록증
   - 없으면 매칭 시스템 이용 불가

3. **등급제로 점진적 신뢰 구축**
   - 처음엔 낮은 등급으로 시작
   - 증빙 추가할수록 등급 상승
   - 후기, 경력 등 복합 평가

4. **자율적 가입 동기 제공**
   - 강제가 아닌 혜택으로 유인
   - "인증하면 더 많은 기회"

## 3. 등급 체계 (4단계)

### 🥉 브론즈 (입문)

**요건:**
- 개인과외교습자 신고필증 **또는** 학원강사 등록증

**혜택:**
- 기본 프로필 등록
- 매칭 시스템 이용 가능
- MathIQ 도구 무료 사용 (학생 관리, 시험 출제 등)

**대상:**
- 과외 시작하는 대학생
- 학원 신입 강사
- 경력 증빙 없는 선생님

---

### 🥈 실버 (검증)

**요건:**
- 브론즈 조건 충족
- 학력 증명서 제출 (졸업증명서 또는 재학증명서)

**혜택:**
- "학력 인증" 뱃지 표시
- 검색 결과에서 브론즈보다 우선 노출

**대상:**
- 학력을 어필하고 싶은 선생님
- 명문대 출신 강조

---

### 🥇 골드 (우수)

**요건:**
- 실버 조건 충족
- 경력 증명서 제출 (재직증명서, 학원 경력 등)
- 학부모 후기 10개 이상 (평균 4.5점 이상)

**혜택:**
- "검증된 선생님" 뱃지
- 프로필 상단 노출
- 매칭 우선권 (같은 조건 시 먼저 표시)

**대상:**
- 3년 이상 경력 강사
- 학부모 평판 좋은 선생님

---

### 💎 다이아몬드 (최고)

**요건:**
- 골드 조건 충족
- MathIQ 자체 실력 테스트 통과

**혜택:**
- "MathIQ 인증 다이아" 마크
- 프리미엄 노출 (메인 페이지 추천 선생님)
- 수수료 우대 (매칭 포인트 할인, 구독 할인)

**대상:**
- 최상위 실력 검증 원하는 선생님
- MathIQ 파트너십 희망자

## 4. 자격증 종류 (한국 법 기준)

### 개인과외교습자 신고필증

| 항목 | 내용 |
|------|------|
| 근거 법률 | 학원의 설립·운영 및 과외교습에 관한 법률 |
| 적용 대상 | 학습자 또는 교습자 자택에서 과외 |
| 발급 방식 | 교육청 신고제 (자격증 아님, 신고만 하면 발급) |
| 학력 요건 | 없음 |
| 비용 | 무료 |
| 유효기간 | 없음 (변경 시 재신고) |

**신고 방법:**
1. 정부24 또는 교육청 방문
2. 신고서 + 신분증 제출
3. 3~5일 내 신고필증 발급

---

### 학원강사 등록

| 항목 | 내용 |
|------|------|
| 근거 법률 | 학원법 |
| 적용 대상 | 학원에서 강의하는 강사 |
| 발급 방식 | 학원을 통해 교육청 등록 |
| 학력 요건 | 대졸 이상 (보통) |
| 비용 | 학원 부담 |
| 유효기간 | 학원 재직 중 유효 |

**등록 방법:**
1. 학원에서 강사 채용
2. 학원이 교육청에 강사 등록 신청
3. 강사 등록번호 발급

---

### 학원 vs 개인과외 비교

| 항목 | 학원 강사 | 개인 과외 |
|------|----------|----------|
| 자격 | 학원강사 등록 | 개인과외교습자 신고 |
| 발급 기관 | 교육청 (학원 통해) | 교육청 (본인 직접) |
| 학력 요건 | 대졸 이상 (보통) | 없음 (신고제) |
| MathIQ 시작 등급 | 실버 가능 (학력 포함) | 브론즈부터 |
| 세금 | 학원이 원천징수 | 본인 신고 |

## 5. 검증 방법

### Phase 1 (초기, 봉쌤 수동)

**현재 계획:**

```
선생님 → 자격증 사진 업로드 → 봉쌤 검토 → 승인/반려
```

- 처리 시간: 1~2일 (영업일 기준)
- 의심 시 추가 자료 요청 (신분증 등)
- 승인 시 등급 부여 + 알림
- 반려 시 사유 안내 + 재신청 가능

**검토 기준:**
- 신고필증/등록증 진위 여부
- 이름·생년월일 일치 여부
- 유효기간 확인 (학원 등록의 경우)

---

### Phase 2 (중기, OCR + AI 보조)

**구상:**

```
선생님 → 자격증 사진 업로드 → OCR 텍스트 추출 → AI 진위 확인 → 봉쌤 최종 확인
```

- OCR로 자격증 번호, 이름, 발급일 자동 추출
- AI가 위조 패턴 감지 (이미지 조작 흔적)
- 봉쌤은 결과만 확인 (처리 시간 단축)

---

### Phase 3 (장기, 교육청 API 연동)

**목표:**

```
선생님 → 신고번호 입력 → 교육청 API 조회 → 자동 검증
```

- 교육청 공공 API 연동 (가능 시)
- 실시간 유효성 확인
- 완전 자동화

**주의:**
- 교육청 API 공개 여부 확인 필요
- 개인정보 동의 절차 필수

## 6. 인증 갱신

### 유효기간

| 등급 | 유효기간 | 갱신 방법 |
|------|----------|----------|
| 브론즈 | 1년 | 자격증 재제출 |
| 실버 | 1년 | 자격증 + 학력 재제출 |
| 골드 | 1년 | 자격증 + 경력 재제출 + 후기 유지 |
| 다이아 | 1년 | 전체 재제출 + 테스트 재응시 |

### 갱신 프로세스

```
만료 30일 전 → 이메일/앱 알림 → 재제출 → 검토 → 갱신 완료
```

### 만료 시 처리

- 만료 후 7일 유예기간
- 유예기간 내 미갱신 시:
  - 매칭 검색에서 제외
  - 프로필 "인증 만료" 표시
  - 기존 매칭은 유지 (진행 중인 수업)

## 7. 수익 모델 연계

### 등급별 매칭 수수료 (Phase 2 예정)

| 등급 | 매칭 시 차감 포인트 | 비고 |
|------|-------------------|------|
| 브론즈 | 500P | 현재 기본값 |
| 실버 | 400P | 20% 할인 |
| 골드 | 300P | 40% 할인 + 우선 노출 |
| 다이아 | 200P | 60% 할인 + 프리미엄 노출 |

### 등급별 구독 할인 (Phase 2 예정)

| 등급 | 학생 관리 구독 | 비고 |
|------|---------------|------|
| 브론즈/실버 | 정가 | - |
| 골드 | 30% 할인 | - |
| 다이아 | 50% 할인 | MathIQ 파트너 |

### 수익 모델 의도

- 등급 올리면 비용 절감 → 인증 동기 부여
- 검증된 선생님에게 유리한 환경 조성
- 플랫폼 신뢰도 상승 → 학부모 유입 증가

## 8. 기술 구현 스케치

### 테이블 설계 (초안)

```sql
-- 인증 정보 테이블
CREATE TABLE tutor_certifications (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  tutor_id UUID REFERENCES profiles(id) ON DELETE CASCADE NOT NULL,
  
  -- 현재 등급
  current_grade TEXT CHECK (current_grade IN ('bronze','silver','gold','diamond')),
  
  -- 자격증 (브론즈 필수)
  cert_type TEXT,           -- 'private_tutor' 또는 'academy_teacher'
  cert_number TEXT,         -- 신고번호/등록번호
  cert_image_url TEXT,      -- 업로드 이미지
  cert_issued_date DATE,    -- 발급일
  cert_verified_at TIMESTAMPTZ,  -- 검증 완료 시점
  cert_verified_by UUID,    -- 검증한 운영자 ID
  
  -- 학력 증명 (실버+)
  edu_type TEXT,            -- 'graduate' 또는 'enrolled'
  edu_school TEXT,          -- 학교명
  edu_major TEXT,           -- 전공
  edu_image_url TEXT,
  edu_verified_at TIMESTAMPTZ,
  
  -- 경력 증명 (골드+)
  career_years INTEGER,     -- 경력 년수
  career_description TEXT,  -- 경력 설명
  career_image_url TEXT,
  career_verified_at TIMESTAMPTZ,
  
  -- 실력 테스트 (다이아)
  test_score INTEGER,       -- 테스트 점수
  test_passed_at TIMESTAMPTZ,
  
  -- 갱신 관리
  expires_at TIMESTAMPTZ,
  renewal_required BOOLEAN DEFAULT false,
  
  -- 메타
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now(),
  
  UNIQUE(tutor_id)
);

-- 인덱스
CREATE INDEX idx_cert_tutor ON tutor_certifications(tutor_id);
CREATE INDEX idx_cert_grade ON tutor_certifications(current_grade);
CREATE INDEX idx_cert_expires ON tutor_certifications(expires_at);
```

### RLS 정책 (초안)

```sql
-- 선생님: 본인 인증 정보 조회/수정
-- 학부모: 선생님 등급만 조회 (상세 정보 X)
-- 운영자: 전체 조회/수정
```

### 파일 업로드

- Storage 버킷: `tutor-certifications` (비공개)
- 경로: `{tutor_id}/{cert_type}_{timestamp}.jpg`
- 서명 URL로 운영자만 열람

## 11. 사회적 영향 (Vision)

### 이 시스템이 정착되면

1. **학부모 신뢰 형성**
   - "MathIQ 인증 = 합법 + 검증" 인식
   - 선생님 선택 시 명확한 기준 제공

2. **무자격 과외 감소**
   - 인증 없으면 매칭 불가
   - 합법 과외만 플랫폼에 노출

3. **우수 선생님 보상**
   - 자격 있는 분이 정당한 평가
   - 경력·실력 어필 가능

4. **사교육 시장 투명화**
   - 세금 신고 유도 (합법 선생님만)
   - 불법 과외 시장 축소

### 장기 비전

> "MathIQ에서 인증받은 선생님 = 믿을 수 있는 선생님"이라는 사회적 인식 형성.
> 학부모가 과외 선생님 찾을 때 가장 먼저 MathIQ를 떠올리게.
