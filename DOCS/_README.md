# 📖 옵시디언 사용 가이드

> ⚠️ **새 세션이 되면 `_현재진행중.md를 읽고 시작하자`라고 지시하고 시작하면 됩니다.**

---

## 🗂️ 폴더 가이드 — 어디에 뭘 넣을지

### `_archive/`
**옛날 버전, 백업** — 안 쓰지만 혹시 몰라 보관
- 예: `CLAUDE-md-original-2026-04-27.md` (압축 전 백업)
- 예: `daily-2026-04-26-original.md` (분리 전 원본)

### `_pending/`
**클로드 코드한테 줄 작업 지시서**
- 형식: `CC-instruction-YYYY-MM-DD-주제.md`
- 한 번 쓰고 끝나는 거 (사용 후 archive로 이동 가능)

### `daily/`
**매일 일지** — 그날 한 일, 떠오른 생각
- 형식: `YYYY-MM-DD.md`
- 가볍게. 정리 안 해도 OK

### `features/`
**제품 기능별 문서** — student-management, accounting 등
- 한 기능당 파일 1개
- 영구 보관, 베타 후 진짜 매뉴얼로 발전 예정

### `getting-started/`
**사용자용 가이드** — 가입, 역할, 첫 단계
- 봉쌤이 아니라 **앱 사용자가 보는 거**
- 베타 후 도움말 페이지로

### `sessions/`
**작업 세션 기록 + 임시 메모**
- 점검 결과, 분류 트리, 정책 메모 등
- 영구 보관할 가치는 있지만 daily보다 무거운 거

### 루트 (폴더 밖)
**핵심 문서 4개만**
- `decisions.md` — 정책 결정 (가장 중요)
- `roadmap.md` — 할 일 목록
- `changelog.md` — 변경 이력
- `faq.md` — 자주 묻는 질문

---

## 📌 새 파일 만들 때 결정 트리

```
이 파일이 뭐예요?
│
├─ 클로드 코드한테 줄 지시서? → _pending/
├─ 오늘 한 일 메모? → daily/
├─ 제품 기능 설명? → features/
├─ 앱 사용자용 가이드? → getting-started/
├─ 한 번 작업한 결과/메모? → sessions/
├─ 정책 결정 사항? → decisions.md에 추가 (새 파일 X)
├─ 변경 이력? → changelog.md에 추가 (새 파일 X)
└─ 옛날 버전 백업? → _archive/
```

---

## ✅ ADHD 친화 규칙

1. **새 파일 만들기 전에 1초 멈추기** — 위 결정 트리 보기
2. **루트에 새 파일 X** — 핵심 4개 외에 만들지 말기
3. **파일명은 영어 또는 짧은 한국어** — 검색 잘 되게
4. **날짜 포함하면 형식 통일**: `YYYY-MM-DD-주제.md`
5. **헷갈리면 sessions/ 에 일단 넣기** — 나중에 옮기면 됨

## 💡 새 세션 시작 의식 (가장 중요)

**옵시디언 daily note 만들거나 클로드 채팅 새로 열 때:**

```
_현재진행중.md를 읽고 시작하자
```

→ 이거 한 줄이면 클로드가 어제까지 한 거 다 복원. 봉쌤 설명 안 해도 됨.

---

## 🔚 세션 끝낼 때 의식

**작업 멈추기 직전에:**

```
정리하고 끝내자
```

→ 클로드가 자동으로:
1. `_현재진행중.md` 업데이트
2. 다음 액션 정리
3. 마지막 보고

봉쌤이 "어디까지 했지" 기억할 필요 없게 만드는 게 목적.
