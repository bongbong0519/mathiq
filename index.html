왜 파이데이아솔루션인가요
현역 선생님이 설계
교육 현장을 잘 아는 현직 수학 교사가 직접 기획. 이론이 아닌 실전 중심 설계입니다.
AI 유사문제 생성
틀린 문제와 유사한 문제를 AI가 자동 생성. 취약 단원을 반복 학습하여 완전 학습을 달성합니다.
선생님 중심 설계
학생 관리, 시험 등록, 성적 분석까지 선생님 업무 흐름에 맞춘 직관적 인터페이스입니다.
합리적인 가격
비싼 학원 프로그램 없이도 체계적 학습 관리. 소규모 학원과 개인 과외에 최적화된 가격 정책.
요금제 추후 공개
현재 모든 기능을 무료로 체험하실 수 있습니다.
정식 요금제는 준비 중이며, 곧 안내드리겠습니다.

주요 기능

학생별 취약점 분석

시험 성적 데이터를 기반으로 학생별 취약 단원과 문제 유형을 자동 분석합니다. 선생님은 데이터를 통해 개별 맞춤 지도 계획을 세울 수 있습니다.

AI 유사문제 자동 생성

틀린 문제와 유사한 새 문제를 AI가 자동으로 생성합니다. 취약 단원을 집중 반복 학습하여 완전 학습을 달성할 수 있습니다.

시험 등록 및 성적 관리

시험을 등록하고 성적을 입력하면 자동으로 분석 리포트가 생성됩니다. PDF·Word 시험지 내보내기도 지원합니다.

    📁
    업로드된 파일이 없습니다
    🗄️ Supabase 연동 포인트
    파일은 storage.objects 테이블에 저장됩니다. 메타데이터(업로더, 상태, 분류)는 files 커스텀 테이블에 연결하세요. Row Level Security (RLS) 설정으로 운영자만 전체 목록에 접근 가능하도록 제한합니다.
    업로드 데이터 활용 전략
    전국 선생님이 올린 데이터, 이렇게 쓸 수 있어요

    🚀 플랫폼 운영자가 데이터를 활용하는 방법

    전국 선생님들이 업로드한 시험지·분석 파일은 단순 저장에 그치지 않습니다. 집계·분석하면 전국 단위 학습 데이터가 됩니다. 아래 파이프라인과 활용 사례를 참고하세요.

    ⚙️ 데이터 처리 파이프라인
    STEP 01
    수집
    선생님 업로드 → Supabase Storage 저장 → files 테이블에 메타데이터 기록
    STEP 02
    분류 / 태깅
    AI(Claude API) 자동 분류: 단원, 난이도, 문항수, 유형 추출
    STEP 03
    집계 / 분석
    Supabase Edge Function → 지역별/학년별/단원별 통계 생성
    STEP 04
    서비스 적용
    플랫폼 내 추천, 리포트, 비교 분석, 외부 판매 등으로 활용
    📊
    전국 평균 비교 리포트
    학급 평균을 전국/지역 평균과 비교하는 리포트 자동 생성. "우리 반이 전국 수와 연산 평균보다 8점 낮습니다" 형태로 선생님에게 인사이트 제공.
    B2B 프리미엄 기능
    📚
    문항 은행 (Question Bank) 구축
    선생님들이 올린 시험지를 AI로 문항 단위 분류 → 단원·난이도별 문항 DB 구축 → 선생님이 시험 자동 생성할 때 활용 가능.
    핵심 경쟁력
    🗺️
    지역별 학력 격차 분석
    지역별 점수 분포 시각화. 교육청·연구소에 익명화된 집계 데이터 제공하여 수익화 또는 공익 활용. 논문 협력 가능.
    B2G / 연구 협력
    🤖
    AI 취약 단원 예측
    특정 시기·단원에서 점수 하락이 반복되는 패턴 감지 → 선생님에게 "이 단원 집중 수업 권장" 알림 발송. 학습 개입 최적화.
    AI 자동화

    🗄️ Supabase 운영자 쿼리 예시
    📥 전국 단원별 평균 점수 집계
    SELECT unit_name,
      AVG(score) as national_avg,
      COUNT(*) as sample_count
    FROM exam_results
    JOIN exams ON ...
    GROUP BY unit_name
    ORDER BY national_avg ASC;
    📁 미검토 파일 알림 (Edge Function)
    -- Supabase Cron (매일 오전 9시)
    SELECT * FROM files
    WHERE status = 'pending'
      AND created_at < NOW()
        - INTERVAL '2 days';
    -- → 운영자 이메일 발송
    ⚠️ 중요: 개인정보 처리
    전국 집계 시 학생 개인정보는 반드시 익명화 처리하세요. student_id를 직접 사용하지 않고, anonymized_hash 컬럼으로 분리 저장하고 집계에만 활용하는 구조를 권장합니다. Supabase RLS 정책으로 운영자만 전체 데이터에 접근 가능하도록 설정하세요.
